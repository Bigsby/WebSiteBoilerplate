Modules & Angular & Code Higlight

Story

One of these days, I started creating a page comparing various programming languages. The thing grew on its own and, at some point and as usual, I realized I made little decisions that hurt me along the way because "it was just for this".

Overview

This is an exploration on various options for the websites I usually do. These need to:

  • Be SPAs (Single Page Applications), i.e., require routing managent;
  • Have code snippets highlight;
  • Grow from a simple boiler placte I can reuse dynamically.

There are, then, three areas of choice:

  1. Module management - loading component (modules) dinamically
  2. Application and Routing - managing application logic and SPA routing
  3. Code Snippet highlighting - self explainatory

Code and Structure

The code is all in this GitHub Repository.

There are shared components that don't need to change regardless of the options, these are:

  • Styles - .css file(s) in styles folder.
  • Data - .json file(s) in data folder.
  • Angular Templates - .html file(s) in tamplates folder.

The non-shared code consists of separate start .html page for each option, in root, and, in the scripts folder there is a folder for both RequireJs (requirejs) and SystemJS ( systemjs). In each there is a core folder containing shared code for bootraping the required modules (bootstrap.js and config.json). And the, a folder for each of the options that contains a folder app with the application logic and module reference for each options (appBuilder.js and config.json).

For each options (.html of origin), core/bootstrap.js reads core/config.json and the option specific app/config.json to configure, reference, etc. the needed modules. Then calls the options specific app/appBuilder.js for the (AngularJS) applicaiton specific components and logic.

My choice

For now, I'll be going with:

  • SystemJS

    Although there are a lot of (little) things I like most in RequireJS it is clearly, the way of the near future, going along where web (and not only) standards are going but, still, making very good job at maintaining compatibility with other stuff, like RequireJS.

  • highlight.js

    It does miss some features, like line numbers, but it is, by far, the lightest and the one requiring less scafolding.
